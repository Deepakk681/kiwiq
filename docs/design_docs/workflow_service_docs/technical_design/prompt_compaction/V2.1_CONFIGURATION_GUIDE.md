## ContextBudgetConfig

**Location**: `services/workflow_service/registry/nodes/llm/prompt_compaction/context_manager.py`

Controls how available input tokens are distributed across message sections.

### Fields

| Field | Type | Default | Range | Description |
|-------|------|---------|-------|-------------|
| `system_prompt_max_pct` | float | 0.10 | 0.0-1.0 | Maximum % for system messages/tools |
| `summary_max_pct` | float | 0.20 | 0.0-1.0 | Maximum % for compacted summaries |
| `marked_messages_max_pct` | float | 0.10 | 0.0-1.0 | Maximum % for marked messages |
| `recent_messages_min_pct` | float | 0.40 | 0.0-1.0 | **Minimum** guaranteed % for recent |
| `buffer_pct` | float | 0.10 | 0.0-0.2 | Safety buffer (% of total context) |

### Usage Examples

```python
from workflow_service.registry.nodes.llm.prompt_compaction.context_manager import (
    ContextBudgetConfig,
    ContextBudget,
)

# Balanced mode (default)
config = ContextBudgetConfig()

# Tool-heavy mode (more system, less summaries)
config = ContextBudgetConfig(
    system_prompt_max_pct=0.20,  # 20% for tools
    summary_max_pct=0.15,         # 15% for summaries
)

# High-context mode (more history)
config = ContextBudgetConfig(
    summary_max_pct=0.30,         # 30% for summaries
    recent_messages_min_pct=0.30  # 30% for recent (more compaction)
)

# Conservative mode (larger buffer for Anthropic/Gemini)
config = ContextBudgetConfig(
    buffer_pct=0.15  # 15% buffer for estimation-based counting
)

# Create budget
budget = ContextBudget.calculate(
    total_context=128000,
    max_output_tokens=16384,
    config=config
)
```

### Edge Cases

**Percentages sum > 1.0:**
- Sections may overlap
- Priority order: recent > marked > summary > system
- Recent minimum is guaranteed first

**System messages exceed limit:**
- Truncated from beginning
- Latest tools kept
- Tool call sequences preserved (atomic)

**Marked messages exceed limit:**
- Oldest marked messages dropped first
- Tool sequences kept atomic

**Summaries exceed limit:**
- Triggers additional summarization (recursive)
- Multiple summary generations accumulate

## ContextBudget

**Location**: `services/workflow_service/registry/nodes/llm/prompt_compaction/context_manager.py`

Calculated token allocations based on ContextBudgetConfig.

### Calculation Example (128K context)

```
Total context: 128,000 tokens
- Max output: 16,384 tokens
- Safety buffer: 12,800 tokens (10%)
= Available input: 98,816 tokens

Section allocations:
- System: 9,881 tokens (10%)
- Recent: 39,526 tokens (40%)
- Marked: 9,881 tokens (10%)
- Summary: 19,763 tokens (20%)
- Reserved: 19,765 tokens (~20%)

Compaction thresholds:
- Trigger: 79,052 tokens (80% of available)
- Target: 44,467 tokens (45% of available)
```

```python

total_context_limit: int      # Model's context window
```

## PromptCompactionConfig

**Location**: `services/workflow_service/registry/nodes/llm/prompt_compaction/compactor.py`

Main configuration for the prompt compaction system.

### Core Fields

```python
# Enable/disable
enabled: bool = True

# Strategy selection
strategy: CompactionStrategyType = HYBRID  # SUMMARIZATION, EXTRACTIVE, HYBRID

# Budget allocation
context_budget: ContextBudgetConfig = ContextBudgetConfig()

# LLM configuration (v2.1)
llm_config: CompactionLLMConfig = CompactionLLMConfig()

# Strategy-specific configs
summarization: SummarizationConfig = SummarizationConfig()
extraction: ExtractionConfig = ExtractionConfig()
hybrid: HybridConfig = HybridConfig()

# Message preservation
recent_message_count: int = 8              # Keep 8 most recent
preserve_tool_call_sequences: bool = True  # Atomic tool sequences

# Trigger thresholds
trigger_threshold_pct: float = 0.80  # Compact at 80%
warning_threshold_pct: float = 0.70  # Warn at 70%
target_usage_pct: float = 0.50       # Target 50% after
```

### Strategy Selection Guide

| Strategy | When to Use | Pros | Cons |
|----------|-------------|------|------|
| **SUMMARIZATION** | Long conversations, narrative content | Compact, coherent | May lose details |
| **EXTRACTIVE** | Technical docs, specific facts | Preserves exact text | Less compact |
| **HYBRID** | General purpose (default) | Balanced | More complex |

## Strategy Configs

### SummarizationConfig

```python
mode: SummarizationMode = FROM_SCRATCH  # FROM_SCRATCH or CONTINUED
min_messages_for_summarization: int = 5  # Minimum to trigger
```

**Modes:**
- `FROM_SCRATCH`: Summarize all historical messages fresh
- `CONTINUED`: Merge existing summaries with new messages (uses **v3.0 linear batching**)

**v3.0 Note:** The `max_summary_chunks` config is deprecated. v3.0 uses linear batching with equal budget distribution (`target_tokens = budget / num_batches`), eliminating hierarchical merge logic entirely

### ExtractionConfig

#### Basic Extraction Settings

```python
construction_strategy: ExtractionStrategy = EXTRACT_FULL
top_k: int = 5                    # Number of relevant messages
similarity_threshold: float = 0.7  # Minimum relevance score
embedding_model: str = "text-embedding-3-small"  # OpenAI embedding model
```

**Construction Strategies:**
- `DUMP`: Concatenate extracted chunks as-is
- `EXTRACT_FULL`: Expand chunks to full original messages (default, preserves context)
- `LLM_REWRITE`: Use LLM to rewrite chunks into coherent summary

#### Message Chunking (v2.1 - for Large Messages)

**When to use:** Individual messages exceeding 6K tokens (embedding model limit) need to be chunked before embedding.

```python
enable_message_chunking: bool = True                    # Enable chunking for large messages
chunk_size_tokens: int = 6000                           # Max tokens per chunk
chunk_overlap_tokens: int = 200                         # Overlap between chunks (preserves context)
chunk_strategy: str = "semantic_overlap"                # How to chunk
chunk_score_aggregation: str = "max"                    # How to aggregate chunk scores
```

**Chunking Strategies:**
- `semantic_overlap`: Split by paragraphs/sentences with overlap (preserves meaning)
- `fixed_token`: Fixed-size token chunks with overlap

**Score Aggregation:**
- `max`: Take highest score from any chunk (recommended for extraction)
- `mean`: Average all chunk scores
- `weighted_mean`: Weight by chunk size

**Example - Large Message Handling:**
```python
extraction=ExtractionConfig(
    enable_message_chunking=True,
    chunk_size_tokens=6000,          # Each chunk <= 6K
    chunk_overlap_tokens=200,        # 200 token overlap preserves context
    chunk_strategy="semantic_overlap",  # Split by paragraphs
    chunk_score_aggregation="max",   # Best chunk represents message
)
```

#### Chunk Expansion (v2.1)

**Controls how retrieved chunks are expanded back to messages:**

```python
expansion_mode: str = "full_message"       # How to expand chunks
max_expansion_tokens: int = 50000          # Safety limit for expansion
max_chunks_per_message: int = 10           # Limit redundancy per message
```

**Expansion Modes:**
- `full_message`: Expand chunk to full original message (preferred for context)
- `top_chunks_only`: Use retrieved chunks directly (more compact)
- `hybrid`: Full message if fits in budget, else top chunks only

**Example - Expansion Strategies:**
```python
# Maximum context (default)
extraction=ExtractionConfig(
    expansion_mode="full_message",
    max_expansion_tokens=50000,
)

# Compact mode
extraction=ExtractionConfig(
    expansion_mode="top_chunks_only",  # More compact, less context
)

# Adaptive mode
extraction=ExtractionConfig(
    expansion_mode="hybrid",  # Balances context vs tokens
)
```

#### Deduplication (v2.1)

**Two levels of deduplication:**
1. **Within-cycle**: Remove duplicates in single compaction
2. **Cross-cycle**: Track chunks across multiple compactions

```python
deduplication: bool = True                 # Within-cycle deduplication
allow_duplicate_chunks: bool = True        # Allow same chunk across cycles
max_duplicate_appearances: int = 3         # Max appearances total
```

**Example - Strict Deduplication:**
```python
extraction=ExtractionConfig(
    deduplication=True,              # Remove duplicates within each extraction
    allow_duplicate_chunks=False,    # Never allow same chunk twice
)
```

**Example - Allow Important Chunks to Repeat:**
```python
extraction=ExtractionConfig(
    deduplication=True,
    allow_duplicate_chunks=True,     # Allow highly relevant chunks to reappear
    max_duplicate_appearances=3,     # But max 3 times total
)
```

#### Overflow Handling (v2.1)

**When extracted chunks exceed budget:**

```python
overflow_strategy: str = "remove_oldest"   # How to handle budget overflow
```

**Strategies:**
- `remove_oldest`: Drop oldest extractions first (default, safest)
- `reduce_duplicates`: Remove duplicate chunks first, then oldest
- `error`: Raise CompactionError (for debugging)

**Example - Overflow Handling:**
```python
extraction=ExtractionConfig(
    overflow_strategy="remove_oldest",  # Graceful degradation
    # overflow_strategy="error",  # For testing/debugging
)
```

#### Weaviate Storage

```python
store_embeddings: bool = True              # Store in Weaviate ThreadMessageChunks
cleanup_old_threads_days: Optional[int] = None  # Auto-cleanup (e.g., 30, 90 days)
```

**Example - Production Settings:**
```python
extraction=ExtractionConfig(
    store_embeddings=True,
    cleanup_old_threads_days=90,  # Auto-cleanup after 90 days
)
```

#### Advanced Options

```python
extract_salient_points: bool = False       # Extract key facts with LLM (experimental)
use_reranking: bool = False                # Rerank with reranking model (experimental)
rewrite_model: Optional[str] = None        # Model for LLM_REWRITE strategy
```

### HybridConfig

```python
extraction_pct: float = 0.05     # 5% of budget for extraction
extraction_first: bool = True     # Extract before summarizing
```

```python
config = PromptCompactionConfig(
    strategy=CompactionStrategyType.EXTRACTIVE,  # Faster than summarization
```

```python
config = PromptCompactionConfig(
```

## Edge Cases

### Very Large Tool Schemas (>10K tokens)

**Problem**: Tool definitions consume too much system budget.

**Solution**:
```python
config = PromptCompactionConfig(
    context_budget=ContextBudgetConfig(
        system_prompt_max_pct=0.30,  # Allocate more
    ),
    trigger_threshold_pct=0.70,  # Trigger earlier
)
```

### Messages Exceeding Embedding Limit (>6K tokens) - v2.1

**Problem**: Individual messages exceed embedding model's 8K token limit, preventing extraction.

**Solution (v2.1)**: Enable message chunking to split large messages before embedding:
```python
config = PromptCompactionConfig(
    extraction=ExtractionConfig(
        enable_message_chunking=True,     # Split large messages
        chunk_size_tokens=6000,            # Below 8K limit
        chunk_overlap_tokens=200,          # Preserve context
        chunk_strategy="semantic_overlap",  # Smart splitting
        chunk_score_aggregation="max",     # Best chunk wins
    ),
)
```

**How it works:**
1. Large message chunked into ~6K token pieces with overlap
2. Each chunk embedded separately
3. Chunks scored for relevance independently
4. Best chunks retrieved
5. Chunks expanded to full message (if `expansion_mode="full_message"`)

### Extremely Long Single Messages (>100K tokens)

**Problem**: Single message exceeds context window entirely.

**Solution**: Combine v2.1 chunking with summarization:
```python
config = PromptCompactionConfig(
    strategy=CompactionStrategyType.SUMMARIZATION,  # Use summarization
    extraction=ExtractionConfig(
        enable_message_chunking=True,      # Chunk before embedding
        chunk_size_tokens=6000,
    ),
    summarization=SummarizationConfig(
        preserve_structured_outputs=True,  # Keep code/JSON intact
    ),
)
```

**Automatic handling**:
1. Message chunked for embedding (v2.1)
2. Chunks summarized in parallel
3. Summaries combined
4. If still too large, recursive compression

### Rapid Tool Call Sequences

**Problem**: Many tool calls create long atomic sequences.

**Solution**:
```python
config = PromptCompactionConfig(
    preserve_tool_call_sequences=True,  # Keep atomic
    context_budget=ContextBudgetConfig(
        recent_messages_min_pct=0.50,  # More recent space
    ),
    trigger_threshold_pct=0.75,  # Trigger before overflow
)
```

### Mixed Content (Code, Tables, JSON)

**Problem**: Extraction may break structured content.

**Solution**:
```python
config = PromptCompactionConfig(
    strategy=CompactionStrategyType.SUMMARIZATION,  # Preserve structure
    summarization=SummarizationConfig(
        mode=SummarizationMode.FROM_SCRATCH,  # Clean summarization
    ),
)
```

## Performance Characteristics (v2.1 → v3.0)

### Parallel Batch Processing (v2.1+)

When messages are split into multiple batches for summarization:

**Sequential (old)**: `5 batches × 3s = 15s`
**Parallel (v2.1+)**: `max(batch times) ≈ 3s`

**Speedup**: 5-10x for multi-batch scenarios

**Configuration impact**: Automatic - no config needed.

### Linear Batching (v3.0)

**v3.0 Architecture Change**: Switched from hierarchical merging to linear batching.

**Benefits**:
- ✅ **Predictable LLM calls**: Exactly N calls for N batches (no recursive merging)
- ✅ **No infinite loops**: Budget guaranteed by `target_tokens = budget / num_batches`
- ✅ **Simpler**: Removed 100+ lines of hierarchical merge logic
- ✅ **Cheaper**: No additional merge LLM calls

**Configuration impact**: Automatic - `max_summary_chunks` config deprecated.

### v2.3 Tool Sequence Management

**Transparent features** (no config needed):
- **MAX SPAN grouping**: Automatically captures full conversational context within tool sequences
- **Orphaned tool handling**: Automatically fixes tool call pairing errors
- **Tool text conversion**: Automatically converts tool structures for summarization
- **Dynamic budget adjustment**: Automatically expands recent limit when containing latest tool sequences (v2.5)

**Configuration impact**: Fully automatic - works seamlessly with all strategies.

### LLM Provider Selection

```python
config = PromptCompactionConfig(
    llm_config=CompactionLLMConfig(
        default_provider="openai",   # Options: openai, anthropic
        default_model="gpt-4o-mini",  # Fast, cheap
    ),
)
```

**Cost considerations**:
- `gpt-4o-mini`: $0.15/1M input, $0.60/1M output - **Recommended**
- `gpt-4o`: $2.50/1M input, $10.00/1M output - Better quality
- `claude-sonnet-4`: $3.00/1M input, $15.00/1M output - Best quality

## Validation Rules

### Automatic Validation

```python
# This will raise ValueError:
config = PromptCompactionConfig(
    strategy=CompactionStrategyType.HYBRID,
    hybrid=HybridConfig(
        extraction_pct=0.25,  # 25%
    ),
    context_budget=ContextBudgetConfig(
        summary_max_pct=0.20,  # 20% - TOO SMALL!
    ),
)
# Error: extraction_pct (0.25) cannot exceed summary_max_pct (0.20)
```

### Best Practices

1. **Buffer size**: Minimum 5% for OpenAI, 15% for Anthropic/Gemini
2. **Trigger threshold**: Keep between 0.70-0.85
3. **Target usage**: Keep between 0.40-0.60
4. **Recent minimum**: At least 0.30 (30%)
5. **Summary max**: At least 0.15 (15%)

## Troubleshooting

### "Context window exceeded" errors

**Symptoms**: LLM API returns context overflow error.

**Diagnosis**:
```python
# Check current usage vs. limits
current_tokens = count_tokens(messages, metadata)
if current_tokens > budget.max_usage_before_compaction:
    print("Should have triggered compaction!")
```

**Solutions**:
1. Increase buffer: `buffer_pct=0.15`
2. Lower trigger: `trigger_threshold_pct=0.70`
3. More aggressive target: `target_usage_pct=0.40`

### Summaries growing too large

**Symptoms**: `summary_limit` exceeded.

**Solutions**:
1. Increase limit: `summary_max_pct=0.30`
2. Use CONTINUED mode: `mode=SummarizationMode.CONTINUED`

**v3.0 Note**: Linear batching automatically distributes budget across batches (`target_tokens = budget / num_batches`), preventing summary overflow. No recursive summarization needed

### Compaction too slow

**Symptoms**: Compaction takes >10 seconds.

**Solutions**:
1. Use EXTRACTIVE: `strategy=CompactionStrategyType.EXTRACTIVE`
2. Skip embeddings: `store_embeddings=False`
3. Use DUMP strategy: `construction_strategy=ExtractionStrategy.DUMP`
4. Check parallel processing is working (v2.1 automatic)

### Important context lost

**Symptoms**: LLM doesn't remember key information.

**Solutions**:
1. Mark important messages: `message.mark_for_retention=True`
2. Increase marked limit: `marked_messages_max_pct=0.15`
3. Use EXTRACTIVE or HYBRID: Better fact preservation
4. Increase recent count: `recent_message_count=12`

## Migration from v2.0 → v2.1 → v3.0

### Breaking Changes

None - All versions are fully backward compatible.

### New Features in v3.0 (2025)

#### 1. Linear Batching Summarization
- **Replaces**: Hierarchical merging (no more infinite loops)
- **Algorithm**: `target_tokens = budget / num_batches` (equal distribution)
- **Benefit**: Predictable LLM call count (exactly N calls for N batches)
- **Config change**: `max_summary_chunks` deprecated (no longer used)

#### 2. v2.3 Tool Sequence Enhancements (Transparent)
- **MAX SPAN grouping**: Captures full conversational context in tool sequences
- **Orphaned tool handling**: Prevents tool call pairing errors
- **Tool text conversion**: Safe conversion of tool structures for summarization
- **Dynamic budget adjustment**: Expands recent budget for latest tool sequences, atomic trimming to historical if needed (v2.5)

### New Features in v2.1

#### 1. Message Chunking for Large Messages
- **enable_message_chunking**: Split messages exceeding 6K tokens before embedding
- **chunk_size_tokens**, **chunk_overlap_tokens**: Control chunking behavior
- **chunk_strategy**: Semantic vs fixed-token chunking
- **chunk_score_aggregation**: How to aggregate chunk scores

#### 2. Chunk Expansion Strategies
- **expansion_mode**: full_message, top_chunks_only, or hybrid
- **max_expansion_tokens**: Safety limit for expansion
- **max_chunks_per_message**: Limit redundancy

#### 3. Deduplication & Overflow Handling
- **deduplication**: Within-cycle dedup
- **allow_duplicate_chunks**: Cross-cycle duplicate tracking
- **max_duplicate_appearances**: Max times a chunk can appear
- **overflow_strategy**: How to handle budget overflow

#### 4. Additional Enhancements (v2.1)
- **Parallel batch processing**: Automatic 5-10x speedup for summarization
- **LLM configuration**: `llm_config` field for model selection
- **Enhanced token counting**: Better Anthropic API integration
- **Weaviate auto-cleanup**: `cleanup_old_threads_days`

**Note**: v3.0 and v2.3 features work seamlessly with v2.1 features with no additional configuration required.

### Recommended Updates for v2.1

```python
# v2.0 (basic)
config = PromptCompactionConfig()

# v2.1 (recommended - with chunking for large messages)
config = PromptCompactionConfig(
    extraction=ExtractionConfig(

        chunk_size_tokens=6000,            # Below embedding limit
        chunk_overlap_tokens=200,          # Preserve context
        expansion_mode="full_message",     # Maximum context

        # Deduplication

        # Overflow handling
        overflow_strategy="remove_oldest",

        # Weaviate cleanup
        cleanup_old_threads_days=90,       # Auto-cleanup

        default_model="gpt-4o-mini",      # Faster, cheaper
    ),
```

```python
config = PromptCompactionConfig()  # Use defaults - they're good!
```

**Custom Configuration**:
1. Choose strategy (HYBRID recommended)
2. Set context_budget percentages
3. Tune trigger_threshold_pct (0.80 default)
4. Adjust target_usage_pct (0.50 default)
5. Configure strategy-specific settings

**Performance**:
- v3.0 linear batching: Predictable, no infinite loops (automatic)
- v2.1 parallel processing: 5-10x faster (automatic)
- v2.3 tool handling: Transparent, no errors (automatic)
- Use `gpt-4o-mini` for speed/cost
- EXTRACTIVE strategy for throughput
- Disable embeddings if not needed

**Reliability**:
- Increase `buffer_pct` for safety
- Lower `trigger_threshold_pct` to compact earlier
- Use `preserve_tool_call_sequences=True`
- Mark critical messages for retention
